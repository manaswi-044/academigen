01 Product Vision
VISION
AcademiGen transforms the tedious process of creating academic records into a
seamless, AI-driven experience — letting students focus on learning, not
formatting.
1.1 Problem Statement
Every semester, thousands of engineering and science students spend hours — sometimes days —
manually formatting lab records, program documentation, and academic reports. The problem is not the
content; it is the repetitive, error-prone, time-consuming structure.
The Current Pain The Cost
Manual formatting of each program 2–4 hours per record
Inconsistent structure across batches Marks deducted for format errors
Copying boilerplate repeatedly High cognitive load, low productivity
No standard template enforcement Different outputs per student
Lost files, no version history Work lost before submission
1.2 Why This Product Exists
AcademiGen is built on a simple premise: students should spend time understanding programs, not
formatting documents. By automating structure, code formatting, output generation, and export, we
eliminate busywork and standardize academic output.
Real-World Relevance
▸ Engineering colleges in India alone have 3.5M+ students submitting lab records
annually
▸ Average student spends 6–10 hours/semester purely on formatting
▸ Teachers waste time correcting format errors instead of evaluating understanding
▸ Remote/hybrid learning created demand for digital, shareable, consistent records
▸ Institutions adopting digital submissions need automated, uniform document generation

08 Security Considerations
8.1 Code Execution Security
Sandbox Security Measures
▸ Network isolation: Code containers have zero internet access (block all outbound)
▸ Filesystem: Read-only except /tmp, max 10MB disk write
▸ Memory limit: 256MB per execution, OOM-killed if exceeded
▸ CPU limit: Max 2 CPU cores, 10-second execution timeout
▸ Process isolation: Each execution in separate container, destroyed after completion
▸ No system calls: Seccomp profile blocks dangerous syscalls (fork, exec, socket)
▸ Code scanning: Pre-scan for import of os, sys, subprocess before execution
▸ Rate limiting: Max 20 executions per user per hour
8.2 File Upload Security
Upload Security Measures
▸ File type validation: Server-side MIME type check (not just extension)
▸ Size limits: Max 10MB per file upload
▸ Antivirus scan: ClamAV integration for uploaded files
▸ Sandboxed parsing: Parse uploaded PDFs/DOCX in isolated Python process
▸ No executable uploads: Block .exe, .sh, .py uploads in template section
▸ Content security: Strip macros from uploaded DOCX before processing
▸ Storage isolation: User files stored in private S3 buckets with signed URLs
8.3 API & Data Security
API & Data Security Measures
▸ All API keys stored server-side only — never exposed to client
▸ JWT authentication with 1-hour expiry + refresh token rotation
▸ HTTPS enforced everywhere (HSTS headers)
▸ Input sanitization: All user text sanitized before AI API calls
